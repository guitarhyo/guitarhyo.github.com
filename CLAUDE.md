# 프로젝트 가이드

이 파일은 Claude Code가 이 저장소에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

웹 기반 미니 게임 모음 사이트입니다. GitHub Pages로 배포되며, 학습 목적으로 게임마다 다른 프레임워크/라이브러리를 사용해 구현합니다.

- **호스팅**: GitHub Pages
- **목적**: 웹 게임 개발 학습 + 포트폴리오
- **추후 계획**: 완성된 게임은 PWA로 만들어 안드로이드 앱으로도 배포

## 사이트 구조

```
/
├── index.html              # 게임 리스트 (허브 페이지, 순수 HTML/CSS/JS)
├── assets/                 # 허브에서 쓰는 공용 이미지·아이콘
├── games/
│   ├── [게임이름1]/        # 각 게임은 독립된 폴더
│   │   ├── index.html      # 게임 진입점
│   │   └── ...             # 프레임워크별 빌드 결과물
│   └── [게임이름2]/
└── CLAUDE.md
```

- **루트 `index.html`**: 게임 리스트를 보여주는 허브 페이지. 단순한 정적 HTML로 유지(빌드 도구 없음).
- **`games/[게임이름]/`**: 각 게임은 완전히 독립된 폴더. 게임마다 다른 프레임워크를 써도 서로 영향 없음.
- 허브에서 각 게임으로는 상대 경로 링크(`./games/runner/`)로 이동.

## 기술 스택 정책

게임마다 다른 프레임워크를 쓰는 것이 **의도된 학습 목표**입니다. 통일하지 않습니다.

- 게임 1: 순수 HTML/CSS/JS (바닐라)
- 게임 2: Vue 3 + Vite
- 게임 3: React + Vite
- 게임 4: 그 외 (Svelte, Phaser.js 등 자유)

새 게임을 추가할 때마다 어떤 스택을 쓸지 사용자에게 먼저 확인할 것.

## 게임별 빌드/배포 규칙

### 순수 HTML/JS 게임
- `games/[이름]/` 폴더에 `index.html`을 바로 배치. 별도 빌드 없음.

### Vite 기반 게임 (Vue/React/Svelte)
- 개발은 `games/[이름]-src/` 같은 별도 작업 폴더에서 진행.
- `vite.config.js`에 반드시 다음 설정:
  ```js
  base: '/[저장소이름]/games/[게임이름]/'
  ```
- 빌드 결과(`dist/`)를 `games/[게임이름]/`으로 복사하여 배포.
- 또는 `vite.config.js`의 `build.outDir`을 `'../games/[게임이름]'`으로 지정.

### 공통
- 모든 경로는 상대 경로 사용 권장 (절대 경로 `/`로 시작 금지).
- 이미지·폰트 등 에셋도 게임 폴더 안에 두기.

## 허브 페이지(index.html) 규칙

- 의존성 없는 순수 HTML/CSS/JS로 유지.
- 게임 카드 목록 형태. 각 카드는 `./games/[이름]/`으로 링크.
- 새 게임이 추가되면 허브의 카드 목록도 함께 업데이트할 것.
- 모바일에서도 잘 보이도록 반응형으로 작성.

## GitHub Pages 배포

- `main` 브랜치의 루트에서 서빙되는 구조.
- 푸시하면 자동 반영됨.
- HTTPS 기본 지원되므로 PWA 변환도 추후 가능.

## 작업 시 지켜야 할 것

1. **새 게임 추가 시**: 어떤 프레임워크로 만들지 먼저 확인.
2. **base path 항상 확인**: GitHub Pages 서브경로에서 작동해야 하므로 빌드 도구의 base 설정 필수.
3. **허브와 게임 분리 유지**: 허브 페이지에 프레임워크 의존성 추가하지 말 것.
4. **커밋 메시지**: 한국어 또는 영어 자유. 단, 변경한 게임을 명시할 것 (예: `[runner] 점프 물리 수정`).
5. **README 업데이트**: 새 게임을 추가하면 루트 `README.md`에도 항목 추가.

## 학습 메모

이 저장소는 학습이 목적이므로, 새 개념이나 패턴을 도입할 때는 코드에 **왜 그렇게 했는지 짧은 주석**을 남겨주세요. 특히 프레임워크별 차이점이 드러나는 부분(상태 관리, 컴포넌트 구조, 렌더링 방식 등)은 학습 포인트가 되므로 설명을 적극적으로 추가할 것.
